# 📁 نظام شؤون الطلاب SIS (Student Information System)

> **"ذاكرة المدرسة الحية والمتصلة لكل طالب"** 🎓🧠
> معظم أنظمة الطلاب تركز فقط على (الاسم + الصف + الحضور). ❌
> أما نظام SIS الذكي هذا، فيتعامل مع الطالب كـ **كيان متكامل**: (هوية شخصية، ارتباط أسري، سجل أكاديمي زمني، تاريخ انضباط وحضور، وأدوات تعليمية).

---

## 🏗️ القوة المعمارية (Architectural Strength)

| الميزة المعمارية | كيف تعمل؟ | النتيجة الإدارية |
| :--- | :--- | :--- |
| **⏳ تصميم زمني (Time-aware)** | لا يوجد "تكرار بيانات" بين السنوات. كل سنة دراسية لها (قيد + حضور + حالة) مستقلة تماماً. | يمنع أخطاء قاتلة مثل انتقال الطالب من صف لآخر مع بقاء حضوره القديم مختلطاً. |
| **🗂️ فصل ذكي للكيانات** | تمييز إستراتيجي بين: الطالب (Student)، والقيد (Enrollment)، والحضور (Attendance). | هذا هو الفارق بين النظام المدرسي الهاوي ونظام الـ ERP المبني باحترافية. |
| **👨‍👩‍👦 الارتباط الأسري** | دعم كامل لتعدد أولياء الأمور، تحديد ولي الأمر (الأساسي)، وأرقام الطوارئ للحالات العاجلة. | التواصل الفوري والموثوق وحل مشكلات غياب الاتصال. |
| **🛡️ الأمان والموثوقية** | لا يمكن حذف طالب بدون أثر، وكل تغيير في القيد محفوظ وموثق زمنياً. | نظام لا ينسى ولا يضيع حق الطالب أو المدرسة. |

---

## 💎 القيمة التي يحصل عليها المستفيدون

### 🏫 للإدارة
*   **تقارير فورية ودقيقة:** أعداد الطلاب النشطين، المنسحبين، والمتغيبين لحظياً. قرارات مبنية على أرقام حقيقية، لا تقديرات وهمية.

### 👨‍🏫 للمعلم
*   **بيانات نقية:** معرفة الطلاب المسجلين فعلياً لديه فقط، ورصد حضور دقيق بدون تلاعب أو إضاعة وقت الحصة.

### 📞 لشؤون الطلاب
*   **تواصل فعال:** الوصول لولي الأمر الأساسي فوراً، وإرسال تنبيهات الغياب بضغطة زر بدلاً من البحث في الأرشيف الورقي.

## 🚀 المقدمة
الطالب هو محور العملية التعليمية، والسبب الذي أنشئت المدرسة من أجله. نظام شؤون الطلاب (SIS) هو القلب النابض للمدرسة الرقمية، حيث يجمع كل ما يخص الطالب من لحظة دخوله بوابة المدرسة وحتى تخرجه في مكان واحد آمن ومنظم، مما يغني عن عشرات السجلات الورقية المبعثرة.

## 💡 كيف يستخدم المبرمج هذا النظام؟ (SQL Examples)

### 1. جلب بيانات "بطاقة الطالب" الشاملة
```sql
SELECT 
    s.full_name AS student_name,
    g.full_name AS guardian_name,
    g.phone_primary AS guardian_phone,
    se.classroom_id,
    c.name_ar AS class_name
FROM students s
JOIN student_guardians sg ON s.id = sg.student_id
JOIN guardians g ON sg.guardian_id = g.id
JOIN student_enrollments se ON s.id = se.student_id
JOIN classrooms c ON se.classroom_id = c.id
WHERE s.id = 1001 AND sg.is_primary = 1;
```

### 2. قائمة الغياب ليوم محدد (لإرسال الرسائل)
```sql
SELECT 
    s.full_name,
    g.phone_primary,
    sa.attendance_date,
    sa.status_id
FROM student_attendance sa
JOIN student_enrollments se ON sa.enrollment_id = se.id
JOIN students s ON se.student_id = s.id
JOIN student_guardians sg ON s.id = sg.student_id
JOIN guardians g ON sg.guardian_id = g.id
WHERE sa.attendance_date = CURRENT_DATE 
AND sa.status_id = 2 -- (غائب)
AND sg.is_primary = 1;
```
